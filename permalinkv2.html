WHY THIS EXISTS

Walrus gives you permanence.
We give you shareability.

Walrus is brilliant decentralized storage — your files live on a distributed network, permanent and tamper-proof. But the blob IDs it spits out aren't built for humans. You can't drop wAtcbEtCYyCX2gPcAv6z84NLyHKmS9mQ into a tweet, a chat, or a résumé. Perma.link bridges that gap.

THE GAP

Blob IDs aren't URLs

Walrus blob identifiers are cryptographic hashes — durable on chain, invisible to friends. Nobody clicks a 32-character token.

THE FIX

Four characters, forever

We issue a short code — perma.link/xk4r — that resolves to your Walrus blob. Memorable. Shareable. Permanent.

THE PROMISE

No server, no expiry

The file lives on Walrus. The code lives on chain. There's no backend you need to trust. Your link won't die because we shut down.

BLOB STORAGE
01 / 05

Blob Storage

Files live on Walrus' decentralized network.

DEEP DIVE

Blob Storage

Every file is encoded and dispersed across Walrus' network of storage nodes using erasure coding. No single server holds your file — it's reconstructed on demand from redundant shards, meaning no single point of failure and no centralized operator who can take it down.

Once stored, the blob's existence is anchored on-chain. Retrieval doesn't depend on us; it depends on the protocol.

SHORT URL
02 / 05

Short URL

Four characters resolve to any blob.

DEEP DIVE

Short URL

Perma.link maps a human-readable 4-char code to the underlying Walrus blob. The mapping is stored on the Sui chain, which means it's verifiable and resistant to tampering — unlike private shortener databases.

Codes are generated collision-free and reserved forever. You can read the resolver logic yourself — it's open source.

FAST REDIRECT
04 / 05

Fast Redirect

Sub-second code resolution, straight to blob.

DEEP DIVE

Fast Redirect

Edge workers resolve your short code in under 200ms by reading the on-chain index. No cold starts, no database round trips. The response streams straight from Walrus.

Cached resolution layers mean repeat visits are near-instant. Slow networks still work — the blob itself is fetched lazily.

Your bytes never pass through a Perma.link server. Privacy and speed, by default.

01
02 · WALRUS_BLOB

Store

Walrus returns a certified blob ID — a long cryptographic identifier anchored on-chain.

Perma.link writes a short code → blob ID mapping to the resolver contract. The mapping is yours.
